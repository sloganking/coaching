# CLAUDE.md — the copy gate

This repo is the LIVE coaching website (**slkingco.com**, GitHub Pages): **a push deploys immediately.**

**Before editing or committing ANY user-facing text** (page copy, headings, button labels, meta/og descriptions):

1. **Read [`writing-values.md`](writing-values.md)** (this repo) — voice, identity, principles, red flags. Run every changed sentence through its "Tests Before Shipping" checklist.
2. **Read `../RPM/selling-language.md`** (Logan's private RPM repo, sibling directory on his machines) — all principles plus the pre-send checklist. Watch especially:
   - **P11 — stand on ground you actually hold** (no credibility clauses; the motive check)
   - **P13 — state, don't negate** (a negation imports the accusation — born from a line shipped to THIS site on Jul 24, 2026: "a specialty, not the whole practice." Logan caught the defense within the hour.)

   If `../RPM` doesn't exist on this machine, **don't commit copy changes** — say so instead.
3. **Approval gate:** the default is Logan picks the exact wording before it goes live. Ship without his word-for-word pick only when he explicitly delegated the choice ("you choose"). When in doubt, send options and wait.
4. Non-copy changes (CSS, images, structure) don't need the gate — but if a change touches wording at all, it's copy.

Why this file exists: on Jul 24, 2026 a desk session shipped a defensive clause to the live site because the selling principles weren't in its context. Logan's ask: make it impossible for an AI session to touch this site's words without the rules loaded. This file loads automatically in Claude Code; the HTML files carry a matching top-of-file comment for everything else.
